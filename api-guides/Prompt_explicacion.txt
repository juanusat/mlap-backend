*Gestion de roles
ARCHIVO A MODIFICAR: routes_private_seguridad_GestionarRoles.yaml

-Tabla(Listado): ID, Rol, Descripcion, estado(es un swicth. Ej. Active)
-Search(rol): Busca por el nombre del rol(Ej. Rol de Ejemplo 10)
- Boton add: inputs para Nombre de rol, descripcion.
- Boton view: Nombre de rol, descripcion.
- Boton permission: muestra un modal donde se encuentra una matriz en la que se pueden cambiar los permisos de un rol.
Por ejemplo, hay filas para los distintos módulos (Ej. Usuario, Actos Liturgicos, parroquia,etc.)* las columnas con las acciones que puede realizar: ver, añadir, editar, eliminar. Y como se define estos permisos, mediante switches en la matriz(5*5)(E, en la posición 3-3 esta activado el Switch, en la posición 4-4 esta desactivado el swicth y así varia en toda la matriz). Y luego se cancela o guarda según el botón que presiones en el modal.
